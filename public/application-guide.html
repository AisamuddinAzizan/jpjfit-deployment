1. Overview

JPJFit manages participant fitness operations from registration to certification and reporting.

Primary Users

Admin, JPJ Officer, and Health Officer collaborate through role-based access.

Main Outcome

Track pass/fail performance, generate official certificates, and notify participants by email.

Data Flow

Participants -> Sessions -> Health/Fitness Results -> Certificates -> Reports.

2. Functions by Module

What each feature does in day-to-day operations.

Dashboard

Shows participant totals, pass/fail totals, session performance chart, and notifications.

Participants

Create and maintain participant profiles used across all test sessions.

Test Sessions

Schedule sessions, assign location/date, track status, and attach participants.

Health Records

Capture medical screening metrics before or during sessions.

Fitness Results

Record physical performance metrics; system calculates score, classification, and pass/fail.

Certificates

Generate PDF certificates for passing participants and email them individually or in bulk.

Reports

View analytics summary and export CSV/PDF for management reporting.

Landing Content CMS

Admin edits public homepage text and hero slider images from dashboard.

Mail Settings

Configure Gmail/SMTP/Mailtrap credentials and sender identity for outbound emails.

User Management

Admin creates/updates users and roles to control access permissions.

3. How To Use The Application

Recommended standard flow.

Step 1: Admin configures users/roles and mail settings.
Step 2: JPJ Officer creates test sessions (typically Jan and Jul each year).
Step 3: Participants are registered and attached to sessions.
Step 4: Health Officer records health screening data.
Step 5: JPJ Officer records fitness metrics and verifies pass/fail output.
Step 6: Generate certificates for participants with pass results.
Step 7: Send certificate emails (single or Send All Pending).
Step 8: Export reports for management and auditing.
Tip: If email sends fail, check Mail Settings first (active mailer, username, app password, from address).

4. Logic Functions

Core backend functions shown in boxes with how to run and what they return/output.

FitnessScoringService::calculate(array $metrics)

Module: Fitness Results

Calculates total score, classification, and pass/fail based on UKJK metrics.

How to run: Submit POST /fitness-results from the Fitness Results form.
Returns: { total_score, classification, result_status }.

CertificateController@store

Module: Certificates

Generates a certificate PDF for a participant/session pair after pass validation.

How to run: POST /certificates with participant_id and test_session_id.
Returns: Redirect with success/error flash; saves PDF in storage and certificate record in DB.

CertificateController@sendEmail

Module: Certificates

Sends one certificate email to the participant with certificate PDF attachment.

How to run: POST /certificates/{certificate}/send-email from Certificates page.
Returns: Redirect with success/error flash; updates sent status when email succeeds.

CertificateController@sendPendingEmails

Module: Certificates

Bulk sends pending certificate emails and skips invalid/unqualified records.

How to run: POST /certificates/send-pending-emails from "Send All Pending" button.
Returns: Redirect with summary: Sent / Skipped / Failed.

MailSettingsService::applyRuntimeConfig()

Module: Mail Settings

Loads DB mail settings and applies active provider configuration at runtime.

How to run: Automatically on app boot, and after saving PUT /mail-settings.
Returns: No direct response; updates runtime mail.* config for outgoing emails.

LandingPageContentService::all()

Module: Landing CMS

Combines stored landing content with fallback defaults.

How to run: Open home page GET / or admin CMS GET /landing-page-content.
Returns: Key-value map of landing fields used in public page rendering.

ReportController@exportCsv

Module: Reports

Builds report dataset and streams downloadable CSV output.

How to run: GET /reports/export/csv from Reports page.
Returns: CSV file response (download).

ReportController@exportPdf

Module: Reports

Builds report view and returns PDF export file.

How to run: GET /reports/export/pdf from Reports page.
Returns: PDF file response (download/stream based on browser).

7. Best Practices

Recommended operational habits.

Keep session schedule consistent

Use Jan/Jul pattern yearly for clean long-term analytics.

Validate participant emails

Ensure email exists before certificate generation/sending.

Run pending email in batches

Use Send All Pending after confirming mail provider settings.

Use export for audit trail

Generate PDF/CSV reports periodically for records and compliance.
